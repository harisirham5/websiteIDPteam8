| **Core Assistive Feature** | **Functionality** | **User Need Addressed** | **Justification** |
| --- | --- | --- | --- |
| **1. Dual Mode AI Vision System** | Real‑time image capture via OV5640 camera, processed through ESP32‑S3 for either OCR (text recognition) or object identification | Users need to read printed/handwritten text and recognize surrounding objects independently | Enables visually impaired users to access visual information without external assistance; supports autonomy and confidence |
| **2. Audio Confirmation & Feedback** | ESP32‑S3 provides spoken prompts for mode switching (“Text mode activated” / “Object mode activated”) and delivers recognition results via Bluetooth earphones | Users require clear, immediate feedback to know which mode is active and to understand recognition results | Audio‑only feedback ensures accessibility for blind users; eliminates reliance on LEDs or displays |
| **3. Confidence Threshold Decision** | Recognition results compared against accuracy thresholds (≥ 80 % text, ≥ 90 % object); unclear results prompt retry | Users need reliable outputs and guidance when recognition is uncertain | Prevents misinformation; maintains trust in device performance |
| **4. Lightweight, Wearable Design** | Compact assembly (≤ 150 g) with modular components (battery, camera, earphone drivers) | Users need comfortable, wearable design suitable for long‑term use | Ensures ergonomic fit and easy maintenance; supports local assembly and repair |
| **5. Affordable and Locally Sourced Components** | Uses ESP32‑S3, OV5640, Bluetooth drivers, and Li‑Po battery (1000–1500 mAh) available in Malaysia | Users require cost‑effective assistive technology | Reduces dependency on imported devices (e.g., OrCam, Envision); promotes accessibility and local innovation |
