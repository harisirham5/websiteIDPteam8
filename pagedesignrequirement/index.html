Design Requirements

Core assistive features, functional requirements, and engineering specifications for readlens smart glasses.

Core Assistive Features

Based on validated user needs from interviews and accessibility research, readlens integrates the following core design features and functionalities to effectively assist blind and myopic users in navigation and reading tasks across Malaysian environments:

Core Assistive Feature Functionality User Need Addressed Justification
1. Dual Mode AI Vision System Real-time image capture via OV5640 camera, processed through readlens App for either OCR (text recognition) or object identification. Users need to read printed text and recognize surrounding objects independently. Enables blind and myopic users to access visual information without external assistance; supports autonomy and confidence.
2. Real-Time Obstacle Detection & Audio Feedback Distance sensor and camera detect obstacles; ESP32-S3 sends alerts through PAM8403 amplifier and earphone drivers. Users require immediate awareness of nearby obstacles to avoid collisions. Provides safety and spatial awareness; critical for blind navigation in crowded or unfamiliar areas.
3. AI Chatbot Interaction Integrated AI assistant in readlens App allows users to ask follow-up questions about captured scenes. Users want contextual understanding beyond basic detection (e.g., “What’s written on the sign?” or “What object is ahead?”) Enhances engagement and comprehension; bridges gap between assistive hardware and intelligent interpretation.
4. Lightweight, Modular Design Compact assembly (≤ 150 g) with detachable modules for battery, camera, and amplifier. Users need comfortable, wearable design suitable for long-term use. Ensures ergonomic fit and easy maintenance; supports local assembly and repair.
5. Affordable & Locally Sourced Components Uses ESP32-S3, OV5640, PAM8403, and Li-Po battery (1000–1500 mAh) available in Malaysia. Users require cost-effective assistive technology. Reduces dependency on imported devices (e.g., OrCam, Envision); promotes accessibility and local innovation.
6. Audio Confirmation & Feedback ESP32-S3 provides spoken prompts for mode switching (“Text mode activated” / “Object mode activated”) and delivers recognition results via Bluetooth earphones. Users require clear, immediate feedback to know which mode is active and to understand recognition results. Audio-only feedback ensures accessibility for blind users; eliminates reliance on LEDs or displays.
7. Confidence Threshold Decision Recognition results compared against accuracy thresholds (≥ 80% text, ≥ 90% object); unclear results prompt retry. Users need reliable outputs and guidance when recognition is uncertain. Prevents misinformation; maintains trust in device performance.
